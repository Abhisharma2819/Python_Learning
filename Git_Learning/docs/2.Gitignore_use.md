## рҹ“ҳ What is `.gitignore`?

The `.gitignore` file tells Git which files and folders it should **ignore (not track)** in version control.

This helps keep repositories clean and secure.

## рҹҺҜ Why Use `.gitignore`?

---

### 1. рҹҡ« Avoid uploading virtual environments

Virtual environments contain thousands of files and can be recreated easily.

```gitignore
venv/
```

---

### 2. рҹ”җ Protect sensitive information

Never commit secrets like API keys or passwords.

```gitignore
.env
```

### Example `.env` file:

```text
DB_PASSWORD=my_secret_password
API_KEY=abc123
```

---

### 3. рҹ§№ Ignore temporary/cache files

Python automatically generates cache files.

```gitignore
__pycache__/
*.pyc
```

---

## вҡ пёҸ Important Note

`.gitignore` only works for **untracked files**.

### вқҢ Problem Example

```bash
git add .env
git commit -m "Added env file"
```

Then later:

```gitignore
.env
```

рҹ‘ү Git will still track `.env` because it was already committed.

---

## рҹӣ пёҸ How to Fix It

To stop tracking a file already committed:

```bash
git rm --cached .env
git commit -m "Stop tracking .env"
```

Now:

* File stays on your system
* Git stops tracking it

## рҹ”Һ How to Check Ignored Files

### Check normal status:

```bash

```

### Check ignored files:

```bash
git status --ignored
```
