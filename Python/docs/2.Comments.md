## 📘 Overview

Comments in Python are used to add explanatory notes to code. They are ignored during program execution and do not affect program output.

---

## ✨ Why Comments are Used

* Improve readability of code
* Explain logic and structure of codebase
* Help understand complex or tricky logic
* Temporarily disable code during debugging or testing

---

## 🧾 Single-Line Comments

Single-line comments start with the `#` symbol.

### Example

```python
# Printing name
name = "AbhishekSharma"
print(name)
```

---

## 📄 Multi-Line Comments

Python does not have a dedicated multi-line comment syntax. However, there are two common approaches:

---

### 1. Using Multiple `#` Symbols (Recommended)

Each line is treated as a separate comment.

```python
# Python program to demonstrate
# multiline comments

print("Multiline comments")
```

---

### 2. Using Triple Quotes (`'''` or `"""`)

Triple-quoted strings are ignored if not assigned to a variable and are often used as comments or docstrings.

```python
"""
Python program to demonstrate
multiline comments
"""

print("Multiline comments")
```

---

## ⚠️ Important Note

* Triple-quoted strings are **string literals**, not real comments
* Best practice is to use `#` for comments
* Triple quotes are mainly used for **docstrings**

---

## 📌 Summary

* Use `#` for single-line and multi-line comments
* Use triple quotes for documentation (docstrings)
* Comments improve code readability and maintainability
