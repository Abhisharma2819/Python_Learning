## 📘 What are Docstrings?

Docstrings (Documentation Strings) are special strings used to document Python code.

They describe what a **module, class, function, or method** does.

### Key Points

* Written using triple quotes (`'''` or `"""`)
* Placed immediately after the function/class/module definition
* Unlike comments (`#`), docstrings are accessible at runtime
* Can be accessed using:

  * `__doc__`
  * `help()`

---

## 🧪 Basic Example

```python id="k1g7qz"
def greet(name):
    """This function greets the user by their name."""
    return f"Hello, {name}!"
```

---

## 🔍 Accessing Docstrings

### Example

```python id="d9n8ka"
def my_function():
    """Demonstrates triple quotes docstring and does nothing."""
    return None

print("Using __doc__:")
print(my_function.__doc__)

print("\nUsing help():")
help(my_function)
```

### Output

```
Using __doc__:
Demonstrates triple quotes docstring and does nothing.

Using help():
Help on function my_function in module __main__:
my_function()
    Demonstrates triple quotes docstring and does no...
```

---

## 📏 Best Practices for Docstrings

A good docstring should:

* Start with a capital letter
* End with a period (.)
* Have a short summary line first
* Include a blank line after summary (for multi-line docs)
* Clearly describe parameters, return values, and behavior

## 🧾 Types of Docstrings

---

## 1. Triple-Quoted Docstrings (Most Common)

```python id="t6wq2p"
def my_func():
    '''This is a docstring using triple single quotes.'''
    return None
```

---

## 2. Google Style Docstrings

```python id="v2q8zn"
def multiply(a, b):
    """
    Multiply two numbers.

    Args:
        a (int): First number
        b (int): Second number

    Returns:
        int: Product of a and b
    """
    return a * b
```

---

## 3. NumPy Style Docstrings

```python id="p8k1sd"
def divide(a, b):
    """
    Divide two numbers.

    Parameters
    ----------
    a : float
        Dividend
    b : float
        Divisor

    Returns
    -------
    float
        Quotient of division
    """
    if b == 0:
        raise ValueError("Division by zero not allowed.")
    return a / b
```

---

## 4. One-Line Docstrings

```python id="m2x9lp"
def power(a, b):
    """Return a raised to power b."""
    return a ** b
```

---

## 5. Multi-Line Docstrings

```python id="z8n4vc"
def add(a, b):
    """
    Add two numbers.

    Parameters:
        a (int): First number
        b (int): Second number

    Returns:
        int: Sum of a and b
    """
    return a + b
```

## 6. Docstrings in Classes

```python id="c9v1re"
class Employee:
    """
    A class representing an employee.

    Attributes:
        name (str): Employee name
        age (int): Employee age
        salary (float): Employee salary
    """

    def __init__(self, name, age, salary):
        """
        Initialize an Employee object.

        Parameters:
            name (str): Employee name
            age (int): Employee age
            salary (float): Employee salary
        """
        self.name = name
        self.age = age
        self.salary = salary

    def promote(self, amount):
        """
        Increase salary by a given amount.

        Parameters:
            amount (float): Raise amount

        Returns:
            str: Updated salary message
        """
        self.salary += amount
        return f"{self.name} promoted! New salary: {self.salary}"
```

## 🎯 Summary

* Docstrings document Python code
* Written using triple quotes
* Accessible via `__doc__` and `help()`
* Used for functions, classes, modules, and methods
* More powerful than comments

---

## 🚀 Key Difference: Comments vs Docstrings

| Feature        | Comments (`#`) | Docstrings (`""" """`)       |
| -------------- | -------------- | ---------------------------- |
| Runtime Access | ❌ No           | ✅ Yes                        |
| Purpose        | Internal notes | Documentation                |
| Placement      | Anywhere       | Immediately after definition |
